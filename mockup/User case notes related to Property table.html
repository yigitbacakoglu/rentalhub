The first thing to understand about the RentalHub model is that it isn’t an estate agent website. There is no stock of properties to rent that I manage. The purpose of the RentalHub model is to allow tenants and owners to manage a property between themselves communicating and administering the tasks required to do this through the RentalHub system.

For this reason, the ‘Property’  is the central item that all users of the system are associated with.

A property will have attributes that do not change, such as RentalHub ID, Address, Property Type, Number of Bedrooms etc. But it will also have attributes that are changeable such as the Rental amount, Lease commencement and expiry dates, Tenants etc.

Two examples of getting a property into the system.

Firstly, the owner creates a RentalHub profile and inputs the details of the property. They will then have the ability to ask a tenant to create a RentalHub profile and connect to the property (acceptance or creation of a lease through RentalHub).

The second way is when a tenants sends a connection request to the owner. In doing this they would put some basic details about the property into the system (thereby creating an initial property profile). The owner then has the option to complete the profile and create a lease through the system.

It is required for the tenants to be able to initiate the profile creation as this provides a significant additional avenue of promoting the RentalHub system and also gathers data on rental properties that may be useful for other reasons.

That is basically the main user cases that the property profile needs to be created for. If you need to have some part of it explained differently or clarified just let me know.


Notes on Leases.

With properties to owners, there will likely be just a 1 to 1 relationship, however there will be a 1 to many relationship between the the property and tenants. Therefore the design needs to accomodate something like ‘sessions’, (the lease term)  that the tenant is connected to a property.

Therefore the property really is associated with the Lease rather than the tenants, and the tenants are associated with the lease.

This relationship is what is indicated in the diagram.
